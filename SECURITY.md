# Security Policy

Finlynq handles personal financial data. Security is taken seriously.

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

Email: hussein.halawi@gmail.com

Include in your report:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional but appreciated)

You will receive a response within 72 hours acknowledging your report.
We aim to release a fix within 14 days for critical issues.

## Scope

The following are in scope:
- Authentication and session management
- MCP server authentication (Bearer tokens, OAuth 2.1)
- API route authorization and data access controls
- Database query injection
- XSS and CSRF vulnerabilities
- Sensitive data exposure

## Out of Scope

- Vulnerabilities requiring physical access to a self-hosted server
- Social engineering attacks
- Denial of service

Thank you for helping keep Finlynq secure.
